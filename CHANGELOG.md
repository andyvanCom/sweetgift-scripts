# Changelog

Все значимые изменения проекта SweetGift Scripts.

---

# 2026-07-23

## Admin dashboard

- добавлена закрытая административная панель SweetGift;
- панель разрешено встраивать только на `sweetgift.ru` и `www.sweetgift.ru`;
- отображаются заказы, выручка, средний чек, популярные товары и категории;
- доступны состояние pipeline и текст ежедневного отчёта;
- импорт товаров, импорт статей, классификацию и отправку отчёта можно запускать вручную;
- service-role ключ не передаётся в браузер, доступ проверяется серверной функцией.

---

# 2026-07-21

## Product import runtime

- импорт товаров переведён с поштучных запросов на пакетные операции;
- повторяющиеся предложения YML объединяются по `product_key` до пакетной записи;
- количество обращений к базе за один импорт сокращено с более чем 2 000 до нескольких десятков;
- новый запуск закрывает старый прерванный статус `running`;
- успешный импорт успевает записать итоговый статус до лимита выполнения Edge Function.

---

# 2026-07-20

## Product card cold load

- manifest загружается сразу, без ожидания `DOMContentLoaded`;
- loader определяет текущий commit GitHub и загружает manifest и модули по неизменяемому SHA;
- `core` загружается раньше зависимых frontend-модулей;
- RPC выполняются напрямую через REST и больше не ждут загрузки Supabase SDK;
- SEO-блоки повторяют запрос после временной ошибки;
- до ответа базы показывается мгновенный SEO-fallback, а успешный ответ хранится семь дней;
- блок недавних товаров замечает поздно добавленный контейнер Tilda.

## Night pipeline runtime fixes

- исправлена авторизация ежедневного импорта статей после ротации ключа;
- тяжёлые пересчёты удалены из `classify-articles` и оставлены отдельным cron-задачам;
- прерванные запуски больше не остаются навсегда в статусе `running`;
- письмо отправляется одной HTML-частью UTF-8 без повреждения русских символов.

---

# 2026-07-18

## Classification coverage

- ночной pipeline автоматически назначает отсутствующие SEO-темы статей;
- классификатор понимает статьи о цветах и пионах;
- для всех доступных товаров формируются тип, ингредиенты, стиль и ценовой сегмент;
- новые товары больше не остаются без `product_seo_entities` после импорта.

## Anonymous order analytics

- реализован `sweetgift-order-tracker.js` для успешных заказов корзины Tilda;
- добавлена идемпотентная RPC `track_product_order`;
- исключено хранение имён, контактов, адреса и текста поздравления;
- повторная отправка заказа больше не дублирует события покупки;
- frontend-роли лишены прямого доступа к таблице заказов.

## Night pipeline observability

- добавлен рабочий журнал `system_job_logs` и закрытая health RPC;
- импорты товаров и статей записывают статус, длительность и результат каждого запуска;
- daily-импорт статей обрабатывает весь набор изменений;
- исчезнувшие из источников товары и статьи деактивируются;
- ежедневный отчёт проверяет свежесть этапов и сверяет источники с базой;
- успешное завершение больше не объявляется при неполном pipeline.

## Архитектура

SweetGift окончательно перешёл от набора JavaScript-скриптов к собственной платформе поверх Tilda.

Основные принципы:

- единый Loader;
- модульная архитектура;
- предварительный расчёт рекомендаций;
- Supabase как основной backend;
- минимальная нагрузка на frontend.

---

## Статьи

Добавлена автоматическая классификация статей.

Автоматически определяются:

- SEO-тема;
- получатель подарка;
- повод;
- тип подарка;
- стиль;
- возраст;
- ингредиенты.

Автоматические сущности сохраняются отдельно от ручной разметки.

---

## Recommendation Engine

Добавлены рекомендации товаров внутри статей.

Для каждой статьи автоматически рассчитываются:

- наиболее подходящие товары;
- коэффициент релевантности;
- причина выбора;
- позиция в выдаче.

Стандартный лимит — 8 товаров на статью.

---

## SEO

Добавлена автоматическая перелинковка:

- товары → статьи;
- статьи → товары.

SEO-блоки формируются автоматически на основании тематических сущностей.

---

## Backend

Добавлены новые процедуры:

- автоматическая классификация статей;
- пересчёт SEO-блоков карточек товаров;
- пересчёт рекомендаций товаров для статей;
- обновление тематических связей.

---

## Автоматизация

Настроен единый ночной цикл обработки данных.

Последовательность выполнения:

1. импорт каталога;
2. импорт статей;
3. классификация статей;
4. пересчёт SEO-блоков товаров;
5. пересчёт рекомендаций товаров для статей;
6. отправка ежедневного технического отчёта.

Дополнительно каждый час автоматически обновляются популярные списки товаров.

---

## URL Normalization

Добавлена нормализация ссылок.

Перед записью в базу автоматически удаляются:

- editionuid;
- UTM-метки;
- рекламные параметры;
- служебные query-параметры.

Это исключает появление дублей одного и того же товара.

---

## Daily Report

Расширен ежедневный технический отчёт.

Добавлен контроль:

- количества товаров;
- количества статей;
- статей без SEO-тем;
- SEO-сущностей;
- рекомендаций;
- состояния ночной обработки.

---
