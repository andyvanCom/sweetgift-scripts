# SweetGift Database Schema

Документ описывает логическую структуру базы данных платформы SweetGift.

Последнее обновление: 2026-07-10

---

# Назначение

SCHEMA.md является техническим справочником структуры базы данных.

Документ содержит:

- назначение таблиц;
- основные связи;
- использование данных;
- основные поля;
- назначение подсистем.

Документ не содержит SQL-кода, секретов и внутренних алгоритмов.

# Каталог товаров

## products_catalog

### Назначение

Главная таблица каталога.

Используется практически всеми подсистемами проекта.

### Основные поля

- product_key
- title
- url
- category
- category_slug
- price
- old_price
- available
- image
- composition

### Используется

- карточка товара
- каталог
- поиск
- рекомендации
- SEO
- аналитика

---

## product_ingredients

### Назначение

Нормализованный состав товаров.

### Основные поля

- product_key
- ingredient
- tag

### Используется

- похожие товары
- фильтры
- SEO
- рекомендации

---

## product_seo_entities

### Назначение

Автоматически определённые SEO-сущности товара.

### Примеры

recipient

occasion

ingredient

style

product_type

price_segment

age

---

## product_card_seo_blocks

### Назначение

Готовые SEO-блоки карточек товара.

Используются frontend без дополнительных вычислений.

---

### Возможные типы

ingredient

style

age

### Назначение

## article_views_daily

### Назначение

Ежедневые агрегированные просмотры.

### Назначение

на страницах статей.

# Использование Frontend

Frontend использует только заранее подготовленные данные.

Не допускается выполнение сложных вычислений на стороне браузера.

Все рекомендации рассчитываются заранее.

---

# Политика хранения

Постоянно хранятся

- каталог
- статьи
- SEO
- рекомендации
- агрегированная статистика

Могут очищаться

- сырые события
- технические журналы
- диагностические данные
- временные таблицы
